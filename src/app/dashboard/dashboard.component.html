For Advanced Use
  1. package-lock.json
    1. Keep the details of each package, which required dependencies
  2. angular.json
  3. tsconfig.json
    1. Is handles Global Configurations
    2. Typescript configuration files
    3. Targets ecmascripts versions, which needs to be target
  4. tsconfig.app.json
    1. Is handle Typescript Application Configuration
  5. tsconfig.spec.json
    1. Is handles Test Configurations
    2. We can includes new files as well
      Example:
      "files": [
        "src/test.ts",
        "src/polyfills.ts"
      ],
      "include":[
        "src/**/*.spec.ts",
        "src/**/*.d.ts"
      ]
  6. .browserslistrc
    1. Contains all the supported browser details as below
      Command to check Browser Lists [npx browserslist]
      last 1 Chrome version last 1 Firefox version last 2 Edge major versions last 2 Safari major versions last 2 iOS major versions Firefox ESR
  7. karma.conf.js
    1. Contains test file configuration
    2. Shows that which browser, libraries needs to be load
    3. Shows the path details, port details
  8. polyfills.ts
    1. Very important File
    2. This file includes polyfills needed by Angular and is loaded before the app
    3. This file is divided into 2 sections
      1. Browser polyfills. These are applied before loading ZoneJS and are sorted by browsers
      2. Application imports. Files imported after ZoneJS that should be loaded before your main file.
      3. Refer Link:
        https://angular.io/guide/browser-support
